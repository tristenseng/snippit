## Gap Detail: UX-02 (RLS Not Wired)

**Status:** partial
**Reason:** `prisma/rls-policies.sql` defines Row Level Security policies using `current_setting('app.current_user_id', true)`. No code in the application ever sets this variable via `SET LOCAL`. Without `SET LOCAL`, PostgreSQL's `current_setting()` returns `NULL`, causing the policy to silently match no rows — the database-level isolation is inactive.
**Functional impact:** Employee data isolation is enforced at the application layer:
- RoleGuard prevents UI rendering of cross-employee data
- API routes scope queries by `locationId` and `role: EMPLOYEE`
- No API route allows an authenticated employee to query another employee's EmployeeDay records
The RLS layer was intended as defense-in-depth; the application-layer controls work correctly.
**Path to full satisfaction:** Add a Prisma middleware extension in `src/lib/prisma.ts` that runs `SET LOCAL app.current_user_id = '<userId>'` on each transaction, or formally document that application-layer RBAC is the sole isolation mechanism for v1.0 and the RLS policies are deferred infrastructure.
