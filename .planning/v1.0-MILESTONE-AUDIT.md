### Critical Integration Issue

**ISSUE-01 — POST /api/batches/[id]/days/[dayId]/entries — Missing Employee Include**

**Affects:** DATA-03
**File:** `src/app/api/batches/[id]/days/[dayId]/entries/route.ts` — `prisma.employeeDay.create` call (no `include` clause)

After a manager adds a new weight entry, `WeightEntryForm.tsx` appends the API response as an `EmployeeDayEntry`. `WeightEntryRow.tsx` line 68 renders `entry.employee.name`. The POST response omits the `employee` relation — guaranteed TypeError, row crashes.

**Workaround:** Page reload restores entries from SSR (GET includes `employee`). Edit and delete paths unaffected.

**Fix:** Add `include: { employee: { select: { id: true, name: true } } }` to `prisma.employeeDay.create` in the POST handler. One line.
