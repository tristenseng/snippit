## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline item gaps (`gap-1`) |
| sm | 8px | Compact element spacing, badge padding (`px-2 py-1`) |
| md | 16px | Default card padding, section padding (`p-4`, `px-4`) |
| lg | 24px | Section vertical spacing (`py-6`), nav padding |
| xl | 32px | Layout top-level gaps (`space-y-8`) |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing (not used in this phase) |

Exceptions:
- Touch targets: 44px minimum height/width (`min-h-[44px] min-w-[44px]`) — applies to all interactive elements including expand/collapse row buttons. Source: `tailwind.config.js` `minHeight.touch`, `ActionButton.tsx`.
- Dashboard card inner padding: 20px (`p-5`) — matches existing dashboard card pattern. Source: `dashboard/page.tsx`.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`text-sm`) | 400 regular | 1.5 |
| Label / Caption | 12px (`text-xs`) | 600 semibold | 1.4 |
| Heading | 24px (`text-2xl`) | 700 bold | 1.2 |
| Display / Metric | 30px (`text-3xl`) | 700 bold | 1.1 (tabular-nums) |

Notes:
- `text-xs font-semibold uppercase tracking-wider` is the established card label pattern (e.g., "TODAY'S PRODUCTION"). Use this exactly for section labels on the `/performance` page.
- `tabular-nums` is required on all numeric gram values to prevent layout shift as numbers change. Source: `dashboard/page.tsx` line 25.
- Outfit font weights available: 300, 400, 500, 600, 700 — only 400 (regular) and 700 (bold) are used in this phase. 600 (semibold) is used for labels and nav links.

## Color

| Role | Tailwind Token | Hex | Usage |
|------|---------------|-----|-------|
| Dominant (60%) | `stone-50` | #FAFAF9 | Page background (`bg-stone-50`) |
| Secondary (30%) | `white` + `stone-200` border | #FFFFFF / #E7E5E4 | All cards, nav bar, drawer (`bg-white border-stone-200`) |
| Accent (10%) | `emerald-600` | #059669 | Reserved for specific elements only (see below) |
| Destructive | `red-600` | #DC2626 | Destructive actions only — not present in this phase |

Accent (`emerald-600`) reserved for:
- Active nav link text and background (`text-emerald-700 bg-emerald-50`) in desktop and mobile nav. Source: `MobileNav.tsx`.
- Primary CTA buttons (`bg-emerald-600`). Source: `ActionButton.tsx` `primary` variant.
- Text links on cards (e.g., "View batch →"). Source: `BatchCard.tsx`.
- Focus rings on interactive elements (`focus:ring-emerald-500`). Source: `ActionButton.tsx`.
- Accent is NOT used on gram metric values — those use `text-stone-900`.
- Accent is NOT used on the `/performance` page data values — raw numbers use `text-stone-900`.

Supporting neutral tokens in active use:
- `text-stone-900` — primary text, metric values, headings
- `text-stone-700` — secondary labels within cards
- `text-stone-600` — supporting body text, subheadings
- `text-stone-500` — muted / tertiary text, empty states
- `text-stone-400` — card section labels (uppercase caps), ghost icon buttons

## Interaction Contract

### Dashboard Production Card

- The card (`bg-white rounded-xl border border-stone-200 p-5`) becomes a `<Link href="/performance">` wrapper.
- On hover: `hover:border-stone-300 hover:shadow-sm transition-all duration-200` — matches `BatchCard.tsx` hover pattern.
- Card content:
  - Line 1: `text-xs font-semibold text-stone-400 uppercase tracking-wider` — label "TODAY'S PRODUCTION"
  - Line 2: `text-3xl font-bold text-stone-900 mt-2 tabular-nums` — total grams value (e.g., `842g`)
  - Line 3: `text-sm font-medium text-stone-600 mt-1` — date + day context (e.g., `March 24 · Day 3`)
- Touch target: the entire card is the tap target. No separate button needed.

### `/performance` Page Layout

Three sections stacked vertically with `space-y-6` between them:

1. **Recent Batch Section** — heading `text-lg font-semibold text-stone-900 mb-3`, followed by a list of `StrainBreakdownRow` components.
2. **Batch History Cards** — heading same style, followed by `grid grid-cols-1 sm:grid-cols-2 gap-4` of batch summary cards.
3. **Strain Totals Section** — heading same style, followed by a simple list of `strain name: Ng total across all batches`.

### `StrainBreakdownRow` — Expand/Collapse

- The row is a full-width `<button>` with `w-full text-left` for tap affordance.
- Collapsed state shows: `{totalGrams}g` (bold, `text-stone-900`) + `{dateLabel} · Day {dayNumber}` (muted, `text-stone-500`), arranged with `flex items-center justify-between`.
- An expand indicator: chevron icon (inline SVG, `h-4 w-4 text-stone-400`) rotates 180° when expanded using `transition-transform duration-200`.
- Touch target: `min-h-[44px]` on the button. Source: established `touch` token in `tailwind.config.js`.
- Expanded state reveals a `<div className="pl-4 pt-2 pb-3 space-y-1">` with per-strain entries:
  - Each entry: `text-sm text-stone-600` — `{Strain Name}: {amount}g`
- Animation: `overflow-hidden` + conditional render (no CSS transition animation — matches WeightEntryRow precedent of simple show/hide). Source: CONTEXT.md "Claude's Discretion" — simple show/hide, no library.
- Border: `border-b border-stone-100` between rows, `last:border-0`.

### Batch History Cards

Each batch card follows the `BatchCard.tsx` pattern exactly:
- `bg-white rounded-xl border border-stone-200 p-4 flex flex-col gap-3`
- No hover link needed (read-only, no navigation action within this phase).
- Content:
  - `text-base font-semibold text-stone-900` — "Batch #N"
  - `text-sm text-stone-500` — date range (e.g., `Mar 1 – Mar 15`)
  - `text-sm font-semibold text-stone-900 tabular-nums` — total grams (e.g., `4,820g total`)

### Strain Totals Section

Simple unstyled list, no cards:
- Each row: `flex items-center justify-between py-2 border-b border-stone-100 last:border-0`
- Strain name: `text-sm font-medium text-stone-900`
- Total: `text-sm tabular-nums text-stone-600` — `{N}g across all batches`

## Copywriting Contract

| Element | Copy |
|---------|------|
| Dashboard card label | TODAY'S PRODUCTION |
| Dashboard card date line (with data) | `{Month Day} · Day {N}` — e.g., `March 24 · Day 3` |
| Dashboard card date line (no day context) | `Most recent submitted day` |
| Dashboard card empty state heading | (no heading — card label stays "TODAY'S PRODUCTION") |
| Dashboard card empty state body | `No data yet` |
| Performance page heading | `My Performance` |
| Recent batch section heading | `Recent Batch` |
| Batch history section heading | `Batch History` |
| Strain totals section heading | `Strain Totals` |
| Strain totals item format | `{Strain Name}: {N,NNN}g total across all batches` — e.g., `Blue Dream: 1,240g total across all batches` |
| Batch history total format | `{N,NNN}g total` |
| Performance page empty state heading | `No performance data yet` |
| Performance page empty state body | `Your data will appear here once your manager submits a day.` |
| Error state (query fails) | `Could not load your performance data. Try refreshing the page.` |
| Destructive confirmation | None — this phase has no destructive actions |

Number formatting: use `toLocaleString('en-US')` for gram values above 999 to produce comma separators (e.g., `1,240g`). Values below 1000 display without separator.

Date format: `Date.toLocaleDateString('en-US', { month: 'long', day: 'numeric' })` — produces `March 24`. Source: RESEARCH.md Pattern 1 / Don't Hand-Roll.

## Accessibility

- All `<button>` elements that are icon-only or expand-toggle must have `aria-label` or `aria-expanded` attributes.
- `StrainBreakdownRow` expand button: `aria-expanded={expanded}` + `aria-controls` pointing to the strain list `id`.
- Gram metric values use `tabular-nums` to prevent layout reflow.
- `touch-action: manipulation` is set globally in `globals.css` — no additional touch handling needed.
- Minimum touch target size: `min-h-[44px]` on all interactive rows and buttons. Source: `tailwind.config.js`.
- Color contrast: `text-stone-900` on `white` = 16.1:1 (WCAG AAA). `text-stone-500` on `white` = 4.6:1 (WCAG AA). `emerald-600` on `white` = 4.5:1 (WCAG AA).
