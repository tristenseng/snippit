**`src/app/(dashboard)/admin/users/[id]/edit/page.tsx`** — server component with client form:

Server part:
1. Auth check + admin verification
2. Fetch user: `prisma.user.findUnique({ where: { id: params.id }, include: { userLocations: { include: { location: true } } } })`
3. Fetch locations: `prisma.location.findMany({ orderBy: { name: 'asc' } })`
4. 404 if user not found

Client form component (inline 'use client' component or separate file):
- Page title: "Edit User" (24px/600) with back link to `/admin/users`
- Email shown as read-only field: `<input disabled className="bg-gray-100 text-gray-500 cursor-not-allowed ...">`
- Editable fields:
  - Name: `<input type="text">` with label "Full Name", pre-filled
  - Role: `<select>` with EMPLOYEE / MANAGER / ADMIN, pre-selected
  - Location: if EMPLOYEE, multi-select checkboxes; if MANAGER/ADMIN, single select
- All inputs: `text-base sm:text-sm min-h-[44px]`, associated `<label>` elements
- Submit: PATCH `/api/admin/users/${id}` with `{ name, role, locationId, locationIds }`
- "Save Changes" (primary) and "Cancel" (secondary, navigates back)
- On success: InlineAlert success, then redirect to user list
- Error: InlineAlert error "Unable to save. Please check your connection and try again."
- If user is deactivated: show red InlineAlert at top "This account is deactivated. Reactivation is not yet supported."
- Focal point: Role select field (highest-consequence edit)
