**`src/components/days/EmployeeAutocomplete.tsx`** — 'use client':
- Props: `locationId: string`, `excludeIds: string[]`, `onSelect: (employee: { id: string; name: string }) => void`, `disabled?: boolean`
- On mount: fetch `/api/employees/search?locationId=${locationId}` to get all active employees
- Input with `text-base sm:text-sm min-h-[44px]`, placeholder "Search employee by name..."
- Uses `useDebounce(query, 200)` for filtering
- Filters: `name.toLowerCase().startsWith(debouncedQuery.toLowerCase())` + not in excludeIds
- Dropdown: `absolute z-10 w-full bg-white border border-gray-200 rounded-md shadow-lg mt-1`
- Each result: `px-3 py-2 text-sm hover:bg-blue-50 cursor-pointer min-h-[44px] flex items-center`
- No results: `px-3 py-2 text-sm text-gray-400 italic` — text: "No employees match '{query}'. Check spelling or add a new account."
- Keyboard navigation: ArrowUp/ArrowDown move active index, Enter selects, Escape closes
- ARIA: input has `role="combobox"`, `aria-expanded`, `aria-controls`; dropdown has `role="listbox"`; items have `role="option"`, `aria-selected`; input has `aria-activedescendant`
- After selection: input clears, onSelect fires with employee
- When disabled (no strain selected): input is disabled with reduced opacity

**`src/components/days/WeightEntryForm.tsx`** — 'use client':
- Props: `batchId: string`, `dayId: string`, `batchStrains: { id: string; strain: { id: string; name: string } }[]`, `locationId: string`, `entries: EmployeeDay[]`, `isSubmitted: boolean`
- State: selectedBatchStrainId, selectedEmployee, grams, hours, entryList, editingEntryId
- Flow:
  1. Manager selects strain via StrainSelector
  2. Manager searches employee via EmployeeAutocomplete (disabled until strain selected)
  3. Manager enters grams (numeric, max 4 digits before decimal, 1 after, placeholder "0.0") and optional hours
  4. "Add Entry" button (disabled until employee selected AND grams > 0) — calls POST `/api/batches/${batchId}/days/${dayId}/entries`
  5. On success: entry added to list, form resets for next employee (strain stays selected)
- Submit Day: button with inline confirmation — text: "Submit Day {batchDay}? Employees will see this data." with "Submit Day" (primary) and "Keep Editing" (ghost)
- After submit: calls POST `/api/batches/${batchId}/days/${dayId}/submit`, refreshes
- Empty state: heading "No entries for Day {batchDay}", body "Select a strain and search for an employee to add the first weight entry."

**`src/components/days/WeightEntryRow.tsx`** — 'use client':

- Display: employee name | grams | hours (or "---") | edit icon | delete icon

- Delete: ghost icon button, on click shows "Remove?" with "Yes, Remove" / "Keep Entry" text links (inline, not modal)
