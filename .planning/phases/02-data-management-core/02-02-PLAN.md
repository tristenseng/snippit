**`src/app/api/batches/route.ts`** — GET list + POST create:

GET handler:
1. `getServerSession(authOptions)` — 401 if null
2. Check `ROLE_PERMISSIONS[activeRole].canManageBatches` — 403 if false
3. Get manager's locationId from `prisma.user.findUnique({ where: { id: session.user.id }, select: { locationId: true } })`
4. If admin (activeRole === 'ADMIN'), return all batches; otherwise filter by `locationId`
5. `prisma.batch.findMany({ where: { locationId }, include: { batchStrains: { include: { strain: true } }, days: { select: { id: true, batchDay: true, isSubmitted: true } } }, orderBy: { number: 'desc' } })`
6. Return 200 with batch array

POST handler:
1. Session + permission checks (same as GET)
2. Zod schema: `z.object({ strainIds: z.array(z.string()).min(1, "At least one strain required") })`
3. Get manager's locationId — 422 if null
4. Check no ACTIVE batch: `prisma.batch.findFirst({ where: { locationId, status: 'ACTIVE' } })` — 409 if exists
5. Auto-increment: `prisma.batch.findFirst({ where: { locationId }, orderBy: { number: 'desc' }, select: { number: true } })` — next = (last?.number ?? 0) + 1
6. `prisma.batch.create({ data: { locationId, number: nextNumber, status: 'INACTIVE', batchStrains: { create: strainIds.map(strainId => ({ strainId })) } }, include: { batchStrains: { include: { strain: true } } } })`
7. Return 201

Use `activeRole` via: `const activeRole = (session.user as any).activeRole ?? session.user.role`

**`src/app/api/batches/[id]/route.ts`** — GET detail + PATCH update:

GET: Session check, permission check, location scope check (batch.locationId must match manager's locationId, or admin), include batchStrains with strains and days. Return 200 or 404.

PATCH: Allow updating batch status to COMPLETED (set completedAt to `new Date()`), and endDate. Zod schema: `z.object({ status: z.enum(['COMPLETED']).optional(), endDate: z.string().datetime().optional() })`.

**`src/app/api/batches/[id]/activate/route.ts`** — POST activate:

1. Session + permission checks
2. Find batch by id, verify locationId matches
3. Check batch.status === 'INACTIVE' — 409 if not
4. Check no other ACTIVE batch at location — 409 if exists
5. `prisma.batch.update({ where: { id }, data: { status: 'ACTIVE', startDate: new Date() } })`
6. Return 200 with updated batch

**`src/app/api/strains/route.ts`** — GET list + POST create:

GET: Session check, return `prisma.strain.findMany({ orderBy: { name: 'asc' } })`. No location scoping — strains are global.

POST: Session check, canManageBatches permission, Zod: `z.object({ name: z.string().min(1).max(50), description: z.string().max(200).optional() })`. Create and return 201.
